# Лабораторна робота №4

Простий HTTP сервер на `Node.js` для варіанту 6 (`titanic.json`).

## Що зроблено

- сервер написаний на вбудованому модулі `http`
- аргументи командного рядка читаються через `commander`
- файл читається асинхронно через `readFile`
- згенерований XML додатково зберігається у `output.xml` через `writeFile`
- XML формується через `fast-xml-parser`
- є запуск і локально, і через `docker compose`
- сервер підтримує і звичайний JSON-масив, і JSON Lines формат

## Що має робити сервер

Сервер читає `titanic.json`, обробляє query-параметри та повертає XML.

Параметри:

- `survived=true` - показати тільки тих пасажирів, які вижили
- `age=true` - додати поле `Age` у відповідь

Базові поля у відповіді:

- `Name`
- `Ticket`

Додаткове поле:

- `Age` якщо передано `age=true`

## Структура проєкту

- `main.js` - основний код сервера
- `package.json` - залежності та скрипти
- `titanic.json` - вхідний файл з даними
- `Dockerfile` - збірка контейнера
- `docker-compose.yml` - простий запуск через Docker
- `README.md` - інструкція

## Встановлення локально

Потрібно:

- `Node.js` 20+ 
- `npm`

Кроки:

1. Відкрийте термінал у папці проєкту.
2. Встановіть залежності:

```bash
npm install
```

3. Запустіть сервер:

```bash
npm start -- --input ./titanic.json --host 127.0.0.1 --port 3000
```

4. Для режиму розробки з автоматичним перезапуском використайте:

```bash
npm run dev -- --input ./titanic.json --host 127.0.0.1 --port 3000
```

5. Відкрийте у браузері:

```text
http://127.0.0.1:3000/
```

## Запуск через Docker Compose

Потрібно:

- встановлений `Docker`
- встановлений `Docker Compose`

Кроки:

1. Відкрийте термінал у папці проєкту.
2. Запустіть команду:

```bash
docker compose up --build
```

3. Після запуску сервер буде доступний тут:

```text
http://localhost:3000/
```

4. Щоб зупинити сервер:

```bash
docker compose down
```

## Приклади запитів

Усі пасажири:

```text
http://127.0.0.1:3000/
```

Тільки ті, хто вижив:

```text
http://127.0.0.1:3000/?survived=true
```

Усі пасажири з віком:

```text
http://127.0.0.1:3000/?age=true
```

Тільки ті, хто вижив, і показати вік:

```text
http://127.0.0.1:3000/?survived=true&age=true
```

## Як перевірити результат

1. Запустіть сервер.
2. Відкрийте один із URL вище.
3. Переконайтеся, що у відповідь приходить XML.
4. Після запиту в корені проєкту з’явиться або оновиться файл `output.xml`.

## Що важливо знати про `fast-xml-parser`

- пакет дозволяє будувати XML зі звичайного JavaScript-об’єкта
- не потрібно вручну склеювати XML рядками
- пакет сам коректно формує теги
- можна увімкнути красивий форматований вивід

## Коротко по логіці роботи

1. При старті програма перевіряє, чи існує вхідний файл.
2. Якщо файлу немає, виводиться помилка `Cannot find input file`.
3. Після запиту сервер читає файл асинхронно.
4. Якщо передано `survived=true`, записи фільтруються по полю `Survived`.
5. Якщо передано `age=true`, у XML додається поле `age`.
6. Готовий XML повертається в браузер і записується у `output.xml`.
